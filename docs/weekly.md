# 整体进展

本周完成直播竞拍后端的核心链路搭建，项目已从基础接口设计推进到可分模块运行与测试的 Go 后端。
当前系统已落地用户认证、商品管理、直播间、出价排行榜、保证金、订单和支付等模块，仍有部分模块需要完善。

工程侧完成了模块化目录结构、统一响应、错误码、配置加载、MySQL/Redis 接入、定时任务、服务层单测与 agent 测试文档沉淀。

# 分模块进展
## 竞拍与商品模块  @后端
### 本周进展
1. 完成商品模块基础能力，支持商家创建、发布、开始、结束和取消竞拍商品。
2. 设计并实现出价能力：出价请求、Redis 排行榜、出价日志、幂等 key、加价幅度校验和防狙击自动延时策略。
3. 商品竞拍状态使用 Redis 承载实时数据，MySQL 保留商品、规则和出价记录，形成实时读写与持久化记录分层。

### 下周计划
1. 补齐 WebSocket 广播能力，将出价成功、排行榜变化、竞拍结束等事件推送给直播间用户。
2. 验证高并发场景下的出价逻辑和竞拍结束边界场景。

## 直播间与用户模块  @后端
### 本周进展
1. 完成用户注册、登录、当前用户信息获取和资料更新等用户模块的基本操作
2. 支持商家开通直播间、开播、下播、查询直播间列表和详情，以及维护直播间状态，待拍商品列表

### 下周计划
1. 接入直播间 WebSocket 连接管理，补齐在线人数、房间事件流和用户进入/离开状态。
2. 完善直播间与商品队列联动，确保商品发布、取消、竞拍结束时队列状态一致，并补充测试

## 交易闭环模块  @后端
### 本周进展
1. 完成保证金模块设计与实现，支持用户出价前缴纳保证金。
2. 完成订单模块，竞拍结束后可生成待支付订单，支持订单查询、支付、取消和超时过期。
3. 完成支付模块薄封装，目前只支持单一返回 success 状态。

### 下周计划
1. 打通从一个商品上架->发布->开始竞拍->用户出价->竞拍结束->生成订单->支付->退款的全流程。
2. 完善订单补偿任务，确保竞拍结束但订单创建失败时能够自动修复。

## 其他  @后端
