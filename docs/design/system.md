# 抖音电商直播竞拍全栈系统设计

## background

在直播电商生态中，珠宝、二手奢侈品等高价值非标品难以统一定价，而"动态竞拍"能通过强互动与竞技感最大化商品价值。本课题聚焦高并发与实时交互场景，邀请你基于React、WebSocket与Node/Go等全栈技术，结合开源AI模型，打造一个完整的直播竞拍闭环系统。该引擎需实现从竞拍商品上架、自定义规则设定、毫秒级实时出价参与、动态排名计算到最终落锤成交的全链路流转。课题不仅考验参赛者在WebSocket实时通信、状态一致性上的硬核架构能力，更鼓励探索AI在动态定价与竞拍气氛营造中的创新应用。

## target

每一秒都有人在出价、每次落锤都让围观者觉得亏了、每个失败者都想参加下一场

## target audience

客群特征（价格带：数千～数万元）

- 有一定消费能力，但非顶级高净值
- 攀比心强，注重社交货币（"我抢到了限量款"）
- 对"捡漏"敏感，喜欢用低于市价的感觉说服自己
- 决策受情绪和环境影响大，理性分析能力有限

## core

1. 实现功能
2. 优化体验
3. 拍卖的效率更高，定价更充分
4. 舆论和热度最大化

## core fundation

1. 自定义规则设定
2. 探索AI在动态定价与竞拍气氛营造中的创新应用

## Behavioral Economics Operational Strategies

1. 贪小便宜 → 锚定效应定价
2. 心理账户 → 重新定义消费框架
3. 从众心理 → 社会压力可视化
4. 损失厌恶 → 时间压力与稀缺性
5. 赌场思维 → 沉没成本与连续参与
6. 类盲盒机制 → 概率游戏增加留存
