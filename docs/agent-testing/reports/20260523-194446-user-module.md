## 失败项

### 失败场景：注册 account 长度小于接口绑定要求仍成功

- 复现步骤：调用 `POST /api/v1/auth/register`，body 中 account 长度为 2，password 使用合法测试密码。
- 期望结果：HTTP 400，`code=40001`，不创建用户。
- 实际结果：HTTP 200，`code=0`，数据库创建账号 `ab`。
- 相关证据：HTTP 摘要 `register_short_account status=200 code=0`；数据库查询 `account=ab active_count=1`。
- 可能原因：`service.validAccount` 只校验非空和最大长度，未校验最小长度；接口绑定未生效或未覆盖该请求。
- 影响范围：短账号可绕过接口契约进入持久化，破坏账号规则一致性。
- 建议修复点：统一 DTO binding 与 service 校验，补齐 account 最小长度 3。
- 建议新增的回归测试：接口和 service 均覆盖 account 长度 2 返回参数错误。

### 失败场景：注册 password 长度小于接口绑定要求仍成功

- 复现步骤：调用 `POST /api/v1/auth/register`，account 合法，password 长度为 5。
- 期望结果：HTTP 400，`code=40001`，不创建用户。
- 实际结果：HTTP 200，`code=0`，数据库创建短密码测试账号。
- 相关证据：HTTP 摘要 `register_short_password status=200 code=0`；数据库查询短密码测试账号 `active_count=1`。
- 可能原因：`service.validPassword` 只校验非空，未校验最小长度和最大长度；接口绑定未生效或未覆盖该请求。
- 影响范围：弱密码可被注册并落库。
- 建议修复点：统一 DTO binding 与 service 校验，补齐 password 长度 6 到 72。
- 建议新增的回归测试：接口和 service 均覆盖 password 长度 5 和 73 返回参数错误。

### 失败场景：并发重复注册失败请求返回 500

- 复现步骤：8 个并发请求同时注册同一个测试账号。
- 期望结果：最多一个成功，其余请求返回可预期业务错误，例如 HTTP 400 / `code=40001`。
- 实际结果：1 个成功，7 个 HTTP 500 / `code=50001`。
- 相关证据：并发窗口 208ms，`success_count=1`，失败响应均为 internal server error。
- 可能原因：数据库唯一约束冲突未映射为 `errorx.ErrInvalidRequest`，DAO 或 service 将底层 duplicate key 直接上抛为内部错误。
- 影响范围：高并发重复注册时客户端收到内部错误，错误语义不稳定。
- 建议修复点：在 DAO `CreateUser` 或 service `Register` 中识别 duplicate key 并转换为参数/重复账号业务错误。
- 建议新增的回归测试：并发重复注册最多一个成功，其余均返回业务错误码。

## Apifox 对齐偏差

- `RegisterRequest` 组件 schema 要求 `name` 字段，但真实代码 `dto.RegisterRequest` 只有 `account` 和 `password`；路径 `/api/v1/auth/register` 的内联 request schema 又只要求 `account` 和 `password`。建议移除组件中的 `name` required 和属性，或避免使用不一致组件。
- 多个错误响应 example 仍是 `message=invalid bid price`，与用户模块真实错误 `invalid request` / `unauthorized` / `invalid identity` 不一致。
- `/api/v1/users/me` 的 get/put/delete 在 Apifox ref 中 `security=[]`，但真实接口需要 Bearer token。建议补齐鉴权声明。

## 风险和建议

- 当前 binding 与 service 校验存在不一致风险。即使修复 HTTP binding，也建议 service 层保持完整长度校验，避免未来非 HTTP 入口绕过。
- 并发重复注册依赖数据库唯一性可以保证最终状态唯一，但错误码语义需要业务化。
- `RegisterRequest` Apifox 组件与路径内联 schema 冲突，容易误导接口调用方。

## 建议沉淀的回归测试

- 注册 account 长度 2 必须失败，长度 3 必须成功。
- 注册 password 长度 5 必须失败，长度 6 必须成功，长度 73 必须失败。
- 并发重复注册同一账号最多一个成功，其余返回业务错误而非 500。
- Apifox 与代码 DTO 对齐检查：注册请求不得要求 `name`。
