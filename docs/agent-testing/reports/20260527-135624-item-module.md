## Apifox 对齐偏差

- `GET /api/v1/items` 和 `GET /api/v1/merchant/items` 的 `page_size`：Apifox 默认值为 20，当前代码 `normalizeListInput` 默认值为 10，本次 runner 证据为 `page_size=10`。
- `rule.start_price=0`：Apifox 和 HTTP binding 当前要求最小 1；`docs/5-21.md` 与 Service 层允许 0 元起拍，该差异仍需产品/接口规范确认。

## 风险和建议

- 建议明确商品状态动作并发语义：严格单成功、幂等成功，还是只要求最终状态合法。
- 建议补充 `order_created` 结算场景：先通过出价模块产生最高价，再触发过期结算。
- 建议对齐 Apifox 与代码的分页默认值，避免接口消费者按 20 条默认分页实现。
- 建议决定 HTTP 是否支持 0 元起拍；当前代码层和接口绑定层不完全一致。

## 建议沉淀的回归测试

- 单元测试：HTTP binding 与 Service 校验差异的显式用例，尤其是 `start_price=0`。
