## 通过项

- 同一 WebSocket ticket 在 12 个并发握手中只成功 1 次，其余 11 次返回 401。
- ticket 被成功消费后 Redis key 不存在。
- 8 个不同用户并发连接同一 room 均成功，连接稳定期 Redis 在线状态收敛到 8。
- 8 个连接并发 `leave_room` 均成功，最终 Redis `online_users` 和 `online_count` 收敛到 0。
- 所有并发场景均有有效重叠窗口证据。

## 失败项

- 无。

## 跳过项

- 连接断开与业务广播同时竞争未覆盖：本次并发计划锁定 ticket 和 presence 两个 P0 场景。
- 高频 Fanout 与慢连接剔除未覆盖：属于建议新增并发/稳定性计划。

## Apifox 对齐偏差

- 不适用；本报告不是接口契约报告。相关偏差见同批次 ws module 报告。

## 风险和建议

- 当前并发测试是小规模正确性验证，不代表容量结论。
- 建议新增慢连接和高频 Fanout 并发计划，验证慢连接剔除不影响其他连接。
