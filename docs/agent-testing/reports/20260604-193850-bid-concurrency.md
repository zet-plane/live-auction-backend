## 发现和说明

- 一口价竞争中，成交后续请求可能返回两类拒绝：
  - Redis state 已 ended 时返回 `40002 auction has ended`。
  - Service 从 hot state 或 DB 看到非 ongoing 商品时返回 `40001 invalid request`。
- 上述 `40001` 行为符合 `docs/agent-testing/modules/bid.md` 中“商品状态不是 ongoing 时返回 invalid request”的契约，不属于状态不一致。
- 首次 runner 失败是计划中可解释失败码范围过窄导致；并非最终状态对账失败。重跑前已预清理首次执行留下的 4 个 item、7 条 BidLog/Rule 和 30 个测试用户。
